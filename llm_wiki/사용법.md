# LLM Wiki + graphify 사용법

## 🎯 이게 뭐하는 건지

**개인용 "제2의 뇌"** 만드는 시스템. 논문·트윗·코드·메모 뭐든 폴더에 던져놓으면:

1. **graphify**가 자동으로 지식 그래프(개념 간 연결망) 생성
2. **LLM(Claude)** 이 읽고 요약해서 위키 페이지로 정리
3. 나중에 질문하면 그래프+위키 뒤져서 출처까지 달아 답변

Karpathy의 `/raw` 폴더 워크플로우 기반.

---

## 📁 폴더 구조

```
llm_wiki/
├── schema.md          ← 규칙서 (건드리지 말것)
├── 사용법.md          ← 이 파일
├── raw/               ← 여기에 원본 파일 던지기 ⭐
└── wiki/              ← LLM이 자동 생성/관리 (직접 수정 금지)
    ├── index.md       ← 목차
    ├── log.md         ← 활동 기록
    ├── topics/        ← 개념 페이지 (attention, RLHF 등)
    ├── entities/      ← 사람/조직 (OpenAI, Karpathy 등)
    └── sources/       ← 원본별 요약 (raw/와 1:1)
```

`graphify-out/`은 첫 ingest 때 자동 생성됨 (graph.json, GRAPH_REPORT.md, graph.html).

## 🔄 기본 사용 흐름

### 1단계: 소스 넣기

**파일이면** `llm_wiki/raw/`에 드래그

```
raw/
├── attention-is-all-you-need.pdf
├── karpathy-tweet-screenshot.png
└── my-notes.md
```

**URL이면** Claude에게:

> "https://arxiv.org/abs/1706.03762 를 llm_wiki에 ingest 해줘"

Claude가 `graphify add <URL> --dir ./llm_wiki/raw`로 자동 다운로드.

### 2단계: Ingest 요청

파일 넣은 뒤 Claude에게:

> **"raw/에 새 파일 넣었어. ingest 해줘"**

Claude가 자동으로:

1. `/graphify ./llm_wiki --update` 실행 → 그래프 업데이트
2. `GRAPH_REPORT.md` 읽고 영향받은 개념 파악
3. `wiki/sources/<파일>.md` 생성 (요약+핵심)
4. `wiki/topics/`, `wiki/entities/` 페이지 생성/업데이트
5. `log.md`에 기록, `index.md` 갱신

### 3단계: 질문하기

궁금한 거 Claude에게:

> **"위키에서 'attention이 왜 중요한가' 답해줘"**

Claude가:

1. `graphify query "..."` 로 그래프 탐색
2. 관련 위키 페이지 읽기
3. **출처 인용해서** 답변
4. Q&A를 그래프에 저장 (다음 질문 때 재활용)

### 4단계: 주기적 건강 체크

한 달에 한 번 정도:

> **"llm_wiki lint 돌려줘"**

Claude가 모순·고아 페이지·오래된 링크·빠진 개념 찾아서 `wiki/lint-report.md`에 기록.

## 💬 말하는 법 (복붙 템플릿)

| 하고 싶은 것 | 말하는 방법 |
|---|---|
| 파일 추가했음 | "raw/에 파일 넣었어, ingest" |
| URL 추가 | "`<URL>` 을 ingest 해줘" |
| 질문 | "위키에서 '`<질문>`' 답해줘" |
| 점검 | "llm_wiki lint 돌려줘" |
| 특정 개념 설명 | "'attention' 노드 explain 해줘" |
| 두 개념 연결 찾기 | "'RLHF'랑 'PPO' 사이 경로 찾아줘" |

## 📥 지원하는 소스 타입

- **텍스트**: `.md .txt`
- **논문**: `.pdf`
- **코드**: `.py .ts .go .rs .java` 등 (AST 자동 분석)
- **이미지**: `.png .jpg` (Claude Vision으로 차트/다이어그램 해석)
- **영상/음성**: `.mp4 .mp3` (Whisper로 전사)
- **URL**: 트위터, arXiv, 일반 웹페이지, PDF, 이미지 → 자동 감지

## ⚠️ 주의사항

1. **`raw/`는 append-only** — 넣은 파일 절대 수정/삭제 금지 (불변 원본)
2. **`wiki/`도 직접 수정하지 말기** — Claude가 관리. 수정하면 다음 ingest 때 꼬임
3. **첫 소스 넣을 때까지 graphify 돌리지 말기** — 빈 그래프 나옴
4. **한꺼번에 많이 넣으면 효율↑** — 5~10개 모아서 한 번에 "ingest" 요청이 한 개씩보다 빠름

## 🔧 수동 CLI (선택)

Claude 없이 직접 돌리고 싶으면:

```bash
graphify update ./llm_wiki          # 증분 업데이트
graphify query "질문"                # 그래프 검색
graphify explain "노드명"            # 노드 설명
graphify path "A" "B"                # 두 개념 연결 경로
graphify add <URL> --dir ./llm_wiki/raw   # URL 추가
graphify watch ./llm_wiki            # 파일 변경 자동 감시
```

단, **위키 페이지 생성/갱신은 CLI로 안 됨** — LLM이 해야 함.
