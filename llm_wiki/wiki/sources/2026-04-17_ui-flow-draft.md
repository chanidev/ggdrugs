# UI 플로우 와이어프레임 초안

## Summary

GGdrugs 서비스의 전체 UI 플로우를 담은 세로 긴 와이어프레임 이미지. 일반 사용자(메인/상세/마이페이지), 업로더, 관리자 3개 역할의 화면 전환을 순서대로 나열한다. 서울시 자치구 지도를 중심으로 한 메인 페이지에서 시작해 클러스터 클릭·목록 토글·상세·북마크·리뷰·업로드·관리자 승인까지 주요 경로가 포함되어 있다. 원본 요구사항정의서 v5.0의 유스케이스를 화면 단위로 시각화한 초안(draft) 단계 아티팩트이다.

> ⚠ **해석 정정(2026-04-17)**: 원본 이미지에는 "예약 진입 → 결제 확정"으로 보이는 화면이 있으나, v5.0 요구사항에 예약·결제 유스케이스 부재. 해당 영역은 마이페이지(A_500) 캘린더 + 리뷰 작성(A_501)으로 재해석됨(log.md 2026-04-17T10:15).

## Key points

### 1. 메인 페이지 (일반 사용자)
- 서울시 자치구 경계선이 표시된 지도 배경 (Kakao Maps 추정)
- 자치구별 이벤트 클러스터 마커 표시
- 로그인 전/후 상태 분기 존재
- 전체 목록 조회 토글 (목록 보기 ↔ 지도 보기 전환)
- 클러스터 클릭 시 해당 지역 이벤트 결과 목록 노출

### 2. 검색·필터 플로우
- 이벤트 목록에서 필터/취소하기 동작 (취소 = 필터 해제 또는 검색 초기화로 추정)
- 필터 해제 후 목록/지도 뷰로 복귀

### 3. 상세 페이지 + 마이페이지 캘린더/리뷰
> ⚠ 원본 이미지엔 "예약 진입 → 결제"로 보이는 영역이 있었으나, v5.0 요구사항에 예약·결제 유스케이스 부재. 해당 영역은 **마이페이지(A_500) 캘린더 요약 팝업 + 리뷰 작성(A_501)** 으로 재해석됨. 상세는 [topics/event-detail-review-flow.md](../topics/event-detail-review-flow.md) 참조.

- 이벤트 상세 페이지에서 북마크(찜) 진입
- 마이페이지 캘린더 요약 팝업 → '리뷰 작성' 버튼 (이벤트 종료일 이후 활성화)
- 리뷰 작성 화면 (별점 + 텍스트 + 사진 ≤5장)

### 4. 업로더 페이지
- 업로드 작업 리스트 (진행 중/완료 이벤트 목록)
- 업로더 전용 메인 페이지 (자신이 등록한 이벤트 지도)
- 업로드 작성 페이지 (신규 이벤트 등록 폼)
- 승인 상태별 뷰 (pending / approved / revision_requested 추정)

### 5. 관리자 페이지
- 이벤트 승인 대기열
- 상세 검토 후 승인/반려/수정요청 액션

## Open questions / contradictions

- 이미지 해상도가 낮아 개별 화면의 세부 레이블 판독이 어렵다. 상세 필드명은 `DB_설계_명세서_v3.docx`와 요구사항정의서 v5.0을 교차 확인해야 확정 가능.
- "취소하기" 플로우의 의미가 불분명 — 필터 해제인지, 예약 취소인지, 검색 초기화인지 문맥에서 결정 필요.
- 리뷰 쓰기가 예약 페이지 안에 있는지 상세 페이지에 있는지 이미지만으로는 경계가 모호함.
- 업로더의 "1계정 = 복수 역할 토글" 컨셉이 이 와이어프레임에서 어느 지점에 드러나는지 확인 필요 (역할 전환 UI 위치).
- 관리자 라벨 부여(event_vibe) 화면이 명시적으로 보이지 않음 — 별도 화면인지 승인 플로우 내 인라인인지 확정 필요.
