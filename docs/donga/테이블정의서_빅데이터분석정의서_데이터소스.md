## 0. 개요

- **모델 수**: 43개 (Prisma `model`).
- **DBMS**: PostgreSQL + PostGIS (extensions: `postgis`, `pg_trgm`, `unaccent`, `citext`).
- **명명 규칙**: Prisma 모델 = PascalCase, 필드 = camelCase(`@map`으로 snake_case 컬럼 매핑), 테이블 = snake_case(`@@map`).
- **벡터 검색**: Qdrant(v1.13) — 이벤트 임베딩은 Postgres 외부에 적재(스키마 외).
- **표기 주의**: PK = Primary Key, FK = Foreign Key, NN = NOT NULL, U = Unique. CHECK 제약은 Prisma가 표현하지 못하므로 마이그레이션 SQL에서 보강(주석 근거).

### Prisma 타입 → SQL 타입 매핑 규약

| Prisma 타입 / @db | SQL 타입 |
|---|---|
| `String` | `VARCHAR` (길이는 `@db.VarChar(n)` 기준) |
| `String @db.Text` | `TEXT` |
| `String @db.Char(n)` | `CHAR(n)` |
| `String @db.Uuid` | `UUID` |
| `Int` | `INTEGER` |
| `Int @db.SmallInt` | `SMALLINT` |
| `BigInt` | `BIGINT` (autoincrement → `BIGSERIAL`) |
| `DateTime @db.Timestamptz` | `TIMESTAMPTZ` |
| `DateTime @db.Date` | `DATE` |
| `Boolean` | `BOOLEAN` |
| `Float` | `DOUBLE PRECISION` |
| `Decimal @db.Decimal(p,s)` | `NUMERIC(p,s)` |
| `Json @db.JsonB` | `JSONB` |
| `String[]` / `BigInt[]` | `VARCHAR[]` / `BIGINT[]` |
| `Unsupported("geometry(Point,4326)")` | `geometry(Point,4326)` (PostGIS, WGS84) |

## 3. 빅데이터 관점 — 데이터 파이프라인 단서

Alle의 빅데이터 분석은 **외부 뉴스 데이터 적재 → 이벤트 매핑 → 임베딩/벡터 검색 → 검색·취향 로깅** 의 흐름으로 구성된다.

### 3-1. 외부 데이터 소스 (뉴스 기사 매핑)

| 단계 | 테이블/컴포넌트 | 역할 |
|---|---|---|
| 수집 | `news_articles` | 동아(`source_name='donga'` 기본) 등 외부 기사 크롤링 적재. `original_url` UNIQUE로 중복 차단. `crawled_at`/`published_at`/`metadata(JSONB)` 보유 |
| 매핑 | `event_article_mappings` | 이벤트↔기사 N:M. `relevance_score NUMERIC(5,4)` = 관련도(임베딩 유사도 추정). `(event_id, article_id)` UNIQUE. 인덱스 `idx_event_article_event(event_id, relevance_score DESC)`로 이벤트별 상위 기사 조회 |
| 커버리지 | (운영 지표) | approved 이벤트 4,111건 중 1,810건 뉴스 매핑 = **44% 커버리지** (CLAUDE.md 근거) |

### 3-2. 임베딩 / 벡터 검색 (Qdrant)

- **벡터 저장소**: Qdrant v1.13 (docker-compose). 이벤트 임베딩은 **Postgres 외부**에 적재되어 본 스키마(43 model)에는 벡터 컬럼이 없음.
- **PostGIS 좌표**: `events.location_geom geometry(Point,4326)` — 지리 공간 검색(bbox/근접). Prisma client 생성에서 제외, raw query 전용. 응답 lat/lng는 ST_X/ST_Y로 derive.
- **LLM 요약**: `events.ai_summary`(gpt-4o-mini 생성) + `ai_summary_at` + `description_hash(MD5)` — description 변경 시 트리거가 요약 자동 무효화. 임베딩/요약 재생성 파이프라인의 캐시·감사 단서.
- **확장**: `pg_trgm`(유사 텍스트), `unaccent`, `citext` — 텍스트 검색 보조.

### 3-3. 검색 로그 / 행동 데이터

| 테이블 | 분석 활용 |
|---|---|
| `search_logs` | `search_type(filter/chat)` + `search_params(JSONB)` — 필터/자연어 검색 행동 로깅. 파티셔닝은 Phase 3+ 연기(볼륨 대비) |
| `chat_sessions` / `chat_messages` | LLM 이벤트 검색 대화 코퍼스(질의-응답). 추천/요약 품질 eval 입력 |
| `bookmarks` / `reviews(sentiment)` | 관심·만족 신호. sentiment는 리뷰 감성 분석 산출(positive/negative/neutral) |

### 3-4. 취향 프로필 / 개인화

| 테이블 | 역할 |
|---|---|
| `user_taste_profiles` | `(taste_dimension, taste_value)` EAV. 사용자×차원 UNIQUE. 추천 개인화·세그먼트 분석 입력 |
| `photos.ai_tags(JSONB)` | 업로드 사진 AI 태깅 결과 — 콘텐츠 기반 태그 데이터 |
| `mate_indexes` / `mate_evaluations(q1~q4)` | 동행 신뢰 지수 산출 데이터(시간약속/의사소통/분위기/재방문 Likert) |

### 3-5. 파이프라인 요약(논리 흐름)

```
[외부 매체 크롤] → news_articles
                      │  (임베딩 유사도 / 키워드 매칭)
                      ▼
events ──── event_article_mappings (relevance_score)
  │  ai_summary(gpt-4o-mini), location_geom(PostGIS), 임베딩→Qdrant
  ▼
검색: filter(PostGIS bbox + 5필터) / chat(LLM + 벡터 유사도)
  │
  ▼
search_logs · chat_messages · bookmarks · reviews(sentiment)
  │  (행동·만족 신호 집계)
  ▼
user_taste_profiles (개인화) → 추천
```

> **거버넌스 주의(스키마 외 정책)**: LLM은 추천·검색·요약에만 사용한다. 이벤트 승인(approval_logs)·성향 부여(event_vibe_assignments)·업로더 승급(uploader_profiles)·신고 조치(reports)는 모두 관리자(사람) 결정 영역이며 LLM에 위임하지 않는다.
