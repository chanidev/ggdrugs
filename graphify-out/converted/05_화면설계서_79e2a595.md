## 1. 개요
### 1-1. 문서 목적
본 화면설계서는 Alle 서비스의 화면 구조, 화면 간 이동 흐름, 각 화면의 UI 구성요소와 동작을 정의한다. 개발(Frontend/Backend) · 디자인 · QA가 공통으로 참조하는 화면 단일 기준 문서다.
### 1-2. 서비스 개요
Alle는 자연어 처리 기반 이벤트·이슈 지도 검색 및 동행 매칭 서비스다. 사용자는 서울 지도 위에서 이벤트(event)를 탐색하고, 자연어 채팅으로 검색하며, 함께 갈 동행 메이트를 추천받아 실시간 채팅으로 약속을 조율한다.
### 1-3. 사용자 역할 (액터)
모델은 “1계정 = 복수 역할 토글” 방식이다. 일반 사용자 계정에 업로더 역할을 추가하며, 별도 계정을 생성하지 않는다.
### 1-4. 핵심 도메인 용어
event — 모든 행사의 상위 개념 (절대 festival로 대체하지 않는다)
event_type — {festival, expo, symposium, conference}
event_vibe — 이벤트 성향 (관리자 부여 라벨)
companion_type — 방문자 측 인원구성 속성
expected_companion — 업로더 측 예상 동반자 속성
period — 기간 필터 {3m, 6m, all, custom}
이벤트 상태머신: pending → revision_requested → pending(재제출) ; pending → approved → ended(종료일 도래) ; pending → rejected(종결)
필터 5종(고정): 지역 / 기간 / 인원구성 / 이벤트 종류 / 이벤트 성향
LLM은 추천·검색·요약에만 사용한다. 이벤트 승인·라벨 부여·업로더 승급 심사는 관리자(사람)의 판단 영역이다.

## 2. 메뉴 구성 (사이트맵)
Alle
│
├─ [공통 헤더] 로고 락업(Alle · SEOUL) · 빠른검색 · 언어토글 · 알림벨 · 로그인/마이페이지
│
├─ 비로그인 진입
│   ├─ /                       메인 지도 + 채팅 검색 (탐색 가능)
│   ├─ /events/:id             이벤트 상세 (열람 가능, 북마크·리뷰는 로그인 게이트)
│   ├─ /community              커뮤니티 게시판 (열람 가능, 글쓰기 비활성)
│   ├─ /community/posts/:id    게시글 상세 (열람 가능)
│   └─ 로그인                  Kakao / Google OAuth (헤더 버튼)
│
├─ 일반 사용자 (active_role = user)
│   ├─ /                       메인 지도 (필터 5종 · 전체목록 · 채팅 검색)
│   ├─ /events/:id             상세 + 북마크 + 리뷰 작성 + 신고
│   ├─ /me                     마이페이지 (탭: 캘린더·북마크·리뷰·구독·추천)
│   │    └─ /me/profile        프로필 (닉네임 수정 · 메이트지수)
│   ├─ /notifications          알림 목록 (유형별 인라인 액션)
│   ├─ /community              커뮤니티 (글쓰기 활성)
│   │    └─ /community/posts/:id  게시글 상세 (좋아요·댓글·대댓글·신고)
│   ├─ 메이트 매칭
│   │    ├─ /mate/form              메이트 추천 받기 폼 (S-08)
│   │    └─ /mate/recommendations   메이트 추천 목록 (블라인드/리스트) (S-09)
│   ├─ 채팅
│   │    ├─ /chat/request           채팅 신청 (?to=&nickname=) (S-10)
│   │    ├─ /chat/rooms             내 채팅방 목록 (허브) (S-11)
│   │    └─ /chat/rooms/:chatRoomId 실시간 채팅방 (축제선택·약속·강퇴) (S-12)
│   ├─ /evaluate/:appointmentId     메이트 평가 (S-13)
│   └─ /credits                     크레딧 내역 (적립 원장)
│
├─ 업로더 (active_role = uploader 토글, 같은 계정)
│   ├─ /uploader                    업로더 콘솔 (상태별 분기 + 내 이벤트)
│   ├─ /uploader/new                새 이벤트 등록
│   └─ /uploader/events/:id/edit    이벤트 수정
│
└─ 관리자 (active_role = admin)
    └─ /admin                       관리자 콘솔 (탭 6종)
                                    events / upload-review / uploaders /
                                    members / audit-logs / reports
진입 분기 메모 /uploader · /admin 헤더 링크는 해당 역할 보유 시에만 노출된다. 모든 라우트는 단일 평면 정의이며, 접근 가드는 페이지 내부에서 처리하고 서버 403으로 이중 방어한다.

## 3. 플로우차트
### 3-1. (a) 회원가입 / 로그인
[헤더 로그인 버튼]
   ├─ Kakao OAuth  ─┐
   └─ Google OAuth ─┴→ [OAuth 콜백] → 세션 생성 (active_role = user 기본)
                          ↓
                   헤더가 [마이페이지 / 알림] 표시로 전환
                          ↓
            (업로더 희망 시) /uploader 에서 역할 토글 신청
                   ※ 별도 계정 생성 없음 — 동일 계정에 역할 추가
### 3-2. (b) 메인 서비스: 이벤트 검색 → 상세 → 북마크/리뷰
[ / 메인 지도 ]
   ├─ 필터 5종(지역·기간·인원구성·이벤트종류·이벤트성향) 적용 → 핀/목록 갱신
   └─ 채팅독 자연어 질의 → LLM 추천·검색·요약 → 후보 이벤트 제시
        ↓ 핀 클릭 / 목록 항목 클릭
   [ 이벤트 요약 패널 ] → "상세 보기"
        ↓
   [ /events/:id 상세 ]  포스터·개요·설명·미니맵·관련기사·프로비넌스
        ├─ 비로그인 → [ 로그인 게이트 박스 ] → (a) 로그인
        ├─ 로그인  → [ 북마크 버튼 ] 토글 → /me 캘린더·북마크 반영
        └─ 리뷰 작성 → [ 리뷰 섹션 ] 등록 → 크레딧 review_complete 적립
### 3-3. (c) 업로더 이벤트 등록 → 관리자 승인
[ /uploader ] 프로필 없음
   → 승급 신청 폼 (KYC 본인인증 + 서류 첨부) → 제출
        ↓ 관리자 심사
[ /admin · uploaders 탭 ] 승급 승인 / 반려   ←(관리자=사람 판단, LLM 위임 금지)
        ↓ 승인 시
[ /uploader ] approved → "새 이벤트 업로드"
        ↓
[ /uploader/new ] 이벤트 등록 → 상태 = pending
        ↓
[ /admin · upload-review 탭 ] 검토
   ├─ approved            → (종료일 도래 시) ended
   ├─ revision_requested  → /uploader/events/:id/edit 재제출 → pending
   └─ rejected            (종결)
        ↓ 승인된 이벤트
[ /admin · events 탭 ] 관리자 event_vibe 라벨 부여
### 3-4. (d) 동행 메이트 매칭 → 채팅 → 약속
[ /mate/form ] 지역·연령대·국적 입력 + 약관/안전가이드 동의 → 저장(성공 다이얼로그)
        ↓
[ /mate/recommendations ] blind → list 전환 (아바타·닉네임·메이트지수 카드)
        ↓ "채팅 신청"  (또는 커뮤니티 작성자 프로필 모달에서)
[ /chat/request?to=&nickname= ] 상대 메이트지수 확인 → 신청 (24h 만료)
        ↓ [ 알림 ] match_request 수신 → 인라인 수락/거절
   수락 → chat_room 생성
        ↓
[ /chat/rooms ] 목록 → [ /chat/rooms/:id 채팅방 ]
   ├─ 축제 선택 박스(동행 대상 event 지정) + 주관처 연락처
   ├─ 실시간 메시지 + 참여자 아바타
   ├─ "같이 가자" 약속 팝업 → 약속 확정 → 메이트 추천 블라인드 처리
   └─ (방장) 즉시 강퇴 / 강퇴투표
        ↓ 약속 시각(appointedAt) 도래 후
[ 알림 mate_eval "평가하기" ] (또는 커뮤니티 추천 영역 "평가하기")
        ↓
[ /evaluate/:appointmentId?evaluatedUserId=&chatRoomId= ]
   ├─ ① 메이트 평가  ──→  ② 축제(동행 이벤트) 평가  ──→ 제출 완료
   └─ (평가 화면 내 분기) 상대 사용자 차단 / 신고
        ↓
[ /credits ] appointment_complete · mate_eval_complete 크레딧 적립

## 7. 공통 UI 규약 (디자인 시스템 요약)
전체 규약은 디자인 시스템 규약을 기준으로 한다. 본 절은 화면설계 적용에 필요한 핵심만 요약한다.
### 7-1. 브랜드 / 방향
제품명 Alle, 서브 락업 SEOUL. 보이스: 에디토리얼 + 종이 지도 감성(쇼핑몰·티켓 앱 아님).
로고: 라인 모노그램(정사각 액자 + A 획 + 버밀리언 크로스바). 획만 사용, 최소 24px.
방향: 지도 유틸리티 극대화 + editorial 한 드롭. 장식 최소(accent 1색 외 거의 없음).
### 7-2. 색상 (Color)
규칙: accent는 “빨간 핀 언어”로만, semantic 색은 상태 UI 전용. Gradient 전면 금지(깊이는 shadow로만).
### 7-3. 타이포그래피
Pretendard 단일 패밀리(Inter/Roboto 금지 — Noto Sans KR fallback이 weight·spacing 깨짐). 다국어(zh/ja/vi)는 fallback 체인 확장만, 교체 금지.
데이터/숫자: Pretendard 500 + tnum. 코드(드묾): JetBrains Mono.
모듈러 스케일(8px base, 1.250): display 40 / h1 32 / h2 24 / h3 20 / body 16 / sm 14 / caption 12. 한국어 라인하이트 약간 넉넉.
### 7-4. 간격 / 라운드 / 모션
간격 8px base: xs4 · sm8 · md16 · lg24 · xl32 · 2xl48 · 3xl64.
Border radius 하이브리드(uniform bubbly 금지): input 4 / 버튼·폼 8 / 카드·포스터 12 / 모달 16 / 칩·태그 full.
Shadow로 “지도 위에 놓인” 깊이. 핀 그림자 --shadow-pin 버밀리언 aura.
모션 minimal-functional + 시그니처 1회: 지도 핀 클러스터 분해(stagger 50ms, 280ms ease-out) — 재사용 금지.
### 7-5. 금지 패턴
보라 그라디언트 · 3-column icon grid · 뚱뚱한 pill 버튼 · gradient CTA · stock photo hero · 둥글둥글 bubbly · 서체 fallback이 깨진 Inter/Roboto 한글 혼용.
### 7-6. 레이아웃 / 디자인 시스템 컴포넌트
메인 지도(S-01): map 60% + 사이드바 40%. 상세·마이·업로더·관리자: 12-col grid, max-width 1200px, gutter 24px. 모바일(≤640px): list/map 토글 + 하단 시트 3-snap.
신규 소셜 화면(커뮤니티·메이트·채팅 S-06/08/09/11/12 등)은 디자인 시스템 컴포넌트를 채택한다. Alle 정체성(버밀리언·Pretendard·anti-bubbly)은 테마 오버라이드로 유지. 기존 발견·상세·마이·업로더·관리자 화면은 Alle 핸드빌드 유지.
