# 산출물 색인 (Deliverables Index)
과제명: 자연어 기반 이벤트·이슈 지도 검색 및 동행 매칭 서비스 — Alle 작성일: 2026.06.02 작성: Alle 팀 견본: 동아 MX School 최종 프로젝트 산출물 (예시 프로젝트 "Pickey" — KeyBERT 트렌드 키워드 서비스)
본 디렉터리는 Alle 프로젝트의 최종 산출물 5종을 담는다. 격식·구성·목차는 동아 견본을 따르되, 내용은 Alle 프로젝트로 채웠다. 제품명은 사용자 대면 텍스트에서 Alle를 사용한다. 레포·패키지·DB 식별자는 ggdrugs / @ggdrugs/*를 유지한다(rename 없음).

## 2. 동아 견본(Pickey)과의 매핑
동아 MX School 견본 산출물 구성을 Alle 산출물에 1:1로 대응시킨다.
Pickey가 "트렌드 키워드 추출(KeyBERT)" 단일 도메인인 반면, Alle는 이벤트 수집·검색·지도·동행 매칭의 복합 도메인이므로 02·04의 분량이 견본 대비 확장되었다.

## 3. 공통 규약 (전 산출물 적용)
용어 통일(절대): event(절대 festival로 대체 금지) · event_type{festival, expo, symposium, conference} · event_vibe · companion_type(방문자) · expected_companion(업로더) · role{user, uploader, admin} · active_role · period{3m, 6m, all, custom}.
이벤트 상태머신: pending → revision_requested → pending(재제출) ; pending → approved → ended(종료일 도래) ; pending → rejected.
필터 5종 고정: 지역 / 기간 / 인원구성 / 이벤트 종류 / 이벤트 성향.
사용자 역할 3종: 일반 사용자 / 업로더 / 관리자 — "1계정 = 복수 역할 토글" 모델(별도 계정 없음).
LLM 거버넌스: LLM은 추천·검색·요약·번역에만 사용. 이벤트 승인·event_vibe 라벨 부여·업로더 승급 심사 등 관리자(사람) 판단은 위임 금지.
문서 구조: 표지(제목·과제명·작성일) → 변경이력 표 → 본문.
