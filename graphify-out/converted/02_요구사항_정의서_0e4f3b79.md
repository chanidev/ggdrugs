## Ⅰ. 개요
### 1. 분석 목적
본 문서는 Alle(자연어 기반 이벤트·이슈 지도 검색 및 동행 매칭 서비스)의 기능·비기능 요구사항을 정의하고, 액터와 유스케이스를 식별하여 설계·구현·검증의 단일 기준선을 확립함을 목적으로 한다.
Alle는 다음 세 가지 핵심 가치를 제공한다.
자연어 + 지도 기반 이벤트 탐색 — 사용자가 자연어로 발화한 의도를 5종 필터(지역 / 기간 / 인원구성 / 이벤트 종류 / 이벤트 성향)로 매핑하고, 카카오 지도 위에 이벤트를 핀·클러스터로 시각화한다.
이벤트 등록·승인 워크플로 — 업로더가 이벤트를 등록하고 관리자가 심사·라벨링하는 상태머신 기반 콘텐츠 파이프라인을 운영한다.
동행 메이트 매칭 — 같은 이벤트를 함께 갈 동행을 규칙 기반으로 추천하고, 실시간 채팅 → 약속 조율 → 평가 → 크레딧 적립으로 이어지는 소셜 루프를 제공한다.
본 분석은 현재 구현된 코드베이스(BFF API 라우팅, Web 화면, LLM 서비스)와 프로젝트 설계 결정 기록에서 도출한 구현 사실을 근거로 한다. 근거가 없는 항목은 (TBD)로 명시한다.
