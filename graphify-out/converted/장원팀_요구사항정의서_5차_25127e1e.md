# Ⅰ. 개요
## 1. 문서의 목적
본 문서는 LLM 모델 기반 이벤트 큐레이션 커뮤니티 웹 개발 요구사항을 정의한다. 본 요구사항정의서는 사용자 니즈에 맞는 이벤트 제공을 위한 설계 문서를 작성하는 기초가 되며, 사용자 유스케이스를 기반으로 SW가 제공할 기능·비기능 및 화면에 구현되어야 할 요구사항을 기술하고 정의한다.
본 문서는 구체적이며 간결하게 표현된다. 추후 시험이 가능해야 하며, 본 문서를 사용하는 대상은 본 과제의 기획자, SW를 개발하는 개발자, 검증하는 테스터이다. 본 과제의 요구사항 도출 및 개발 과정에서 공통으로 참조하는 기준 문서로 활용한다.
## 2. 요구사항 및 문서의 범위
본 문서에서는 유스케이스 명세 및 기능·비기능 요구사항 기술을 그 범위로 한다. 5차 개정에서는 기존 4차 대비 다음이 추가/변경되었다:
- [신규] A_203 예정 이벤트 조회 유스케이스
- [신규] A_501 이벤트 리뷰 작성 유스케이스 (와이어프레임 6-1 기반)
- [신규] 용어집 섹션 (Ⅴ장)
- [수정] A_100, A_200, A_500, A_600 — 역할 전환 및 비회원 진입 흐름 정합성 보정
- [수정] 필터 조건 표기 통일 (지역/기간/인원구성/이벤트 종류/이벤트 성향)
- [제거] AI 비디오 자동 생성 관련 언급

# Ⅴ. 용어집 [신규]
본 섹션은 문서 및 개발 전반에서 사용되는 용어의 의미를 통일하기 위한 참조 정의이다.

### 1. 이벤트 계층
- 이벤트 (event) — 본 서비스가 다루는 모든 행사의 상위 개념. DB의 최상위 엔티티.
- 이벤트 종류 (event_type) — 이벤트의 1차 분류. {축제, 박람회, 심포지움, 컨퍼런스} 4종. 업로더가 업로드 시 택1.
- 축제 (festival) — 이벤트 종류 중 하나. 수기 기획서의 테스트 케이스 범위.

### 2. 분류·라벨 관련
- 이벤트 성향 (event_vibe) — 이벤트의 분위기·경험적 특성을 나타내는 라벨. 예: {활동적, 정적, 체험형, 관람형, 교육형, 네트워킹 중심} 등. 관리자가 A_700 심사 시 직접 부여.
- 인원구성 (companion_type) — 방문자 측 속성. {혼자, 연인, 친구, 가족}. 필터 조건으로 사용.
- 동행인 유형 (expected_companion) — 업로더 측 속성. 업로더가 A_602 업로드 시 예상 상위 2개 동행인 유형을 선택. DB 레벨에서는 companion_type과 동일 도메인을 공유하지만 의미가 다르므로 컬럼명을 분리 관리.

### 3. 역할(role) 관련
- 일반 사용자 (user) — 이벤트 조회·북마크·리뷰 등 소비 측면 기능을 사용하는 역할. 모든 회원이 기본 보유.
- 업로더 (uploader) — 이벤트를 등록·관리하는 역할. 일반 사용자가 A_600 승급 신청 후 관리자 A_700 승인을 받으면 동일 계정에 추가된다.
- 관리자 (admin) — 이벤트 심사, 라벨 부여, 업로더 역할 심사를 수행. 별도 관리자 전용 계정.
- active_role — 동일 계정에 여러 역할이 있을 때 현재 활성 역할. 마이페이지 'GG-ROLE-001' 토글 버튼으로 전환.

### 4. 기간 필터 관련
- 기간 (period) — 이벤트 개최 시기를 필터링하는 조건. 하위 옵션: {3개월 이내, 6개월 이내, 전체, 커스텀(년월 지정)}.
- 과거 이벤트 — 현재 시점 기준 종료일이 지난 이벤트. 기본 검색에서는 제외되나 마이페이지 캘린더 및 기록에서 조회 가능.
- 예정 이벤트 — 현재 시점 기준 시작일이 미래인 이벤트. A_203 전용 탭에서 조회.

### 5. 이벤트 상태(state) 관련
- 승인 대기 (pending) — 업로더 업로드 직후 상태.
- 보완 요청 (revision_requested) — 관리자가 추가 서류/정보를 요청한 상태.
- 반려 (rejected) — 관리자가 최종 반려한 상태. 사유 기록.
- 업로드 완료 (approved) — 관리자가 승인하여 사용자 검색 결과에 노출되는 상태.
- 종료 (ended) — 개최 종료일이 지난 상태. 검색 노출 제외 (마이페이지 기록에서는 유지).

### 6. 기술 용어
- BFF (Backend For Frontend) — 프론트 클라이언트 전용 중계 백엔드. 본 프로젝트는 Node.js + Express + Prisma로 구현.
- LLM 마이크로서비스 — 채팅 검색, 리뷰 분류, 이미지 태깅 등 AI 기능을 담당하는 별도 서비스. Python FastAPI + LangChain.
- 벡터 검색 — Qdrant에 저장된 임베딩으로 유사 이벤트를 검색하는 기능.
